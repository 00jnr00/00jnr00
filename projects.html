ENGINEERING - 3.6L INLINE 4

Over the course of 2 months I've designed and simulated an inline-4 cylinder internal combustion engine using the following programs: Fusion360, Blender, and this open-source ICE simulation tool.

This design is unique because of its use of a rotary valve in favor of the tradtional poppet valve actuated by camshafts; in principal, the cam shaft and poppet valve have been combined. Utilzing mechanical seals and carbon bushings, the increased airflow and simplicity of the design, have allowed it to produce 900(crank) horspower in simulations. That translate to about 250 horspower per liter of displacement For reference, the current horspower per liter record out of a naturally aspirated engine is held by the Honda S2000's F20C engine, boasting 125hp per/L. PROGRAMMING - OBSIDIANDART

Cloud based password managment is inherently vunreable; violating the software engineers' most fundamental axioms in relation to security. More interfacing software stacks means more points of failure. When it comes to programs which handle senstive data, like password managers, they're one of the last things you want to be vunreable.

ObsidianDart is a dead-simple AES 256-bit encryption program written in C++ by me. Its made for Linux systems and utilizes (usually) pre-packaged encryption tools in your distro. (Namely: OpenSSL) ObsidianDart is opensource, this is a link to the github page. Being operated via the terminal, the program can encrypt and decrypt a file within the local folder(passwords.wf). The user can choose to use tab or comma seperated values and open it within another program to parse said formatting. ObsidianDart is soley a lightweight cryptography tool, intented for senstive, text-based information.
